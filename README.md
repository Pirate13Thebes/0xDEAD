# 0xDEAD | The Python Quest Repository

## 📜 Project Overview

**0xDEAD** is a comprehensive, quest-based Python repository engineered by a collaborative team of five developers. This project serves as a structured journey through the Python language, moving from foundational syntax to advanced algorithmic logic.

The repository contains **30 unique scripts** organized into **6 progressive levels**. Each script represents a "Quest" designed to master specific programming concepts while maintaining high standards for clean code, modularity, and internal documentation.

## 🗺️ Learning Path (The 6 Levels)

### 1\. Fundamentals (Level 1 & 2)

  * **Concepts:** Variables, Data Types (int, float, string), and String Interpolation.
  * **Interactivity:** Handling user input and basic arithmetic operators.

### 2\. Decision Trees (Level 3)

  * **Concepts:** Conditional logic (`if`, `elif`, `else`) and Logical Operators (`and`, `or`, `not`).
  * **Challenges:** Password checkers, bridge guardians, and grade calculators.

### 3\. Loop Automation (Level 4)

  * **Concepts:** `for` loops, `while` loops, and `range()` manipulation.
  * **Challenges:** Rocket countdowns, even number foraging, and secret number riddles.

### 4\. Modular Programming (Level 5)

  * **Concepts:** Function definition (`def`), Parameters, and Return values.
  * **Challenges:** Reusable incantations and calculated area oracles.

### 5\. Grand Challenges (Level 6)

  * **Concepts:** Algorithmic thinking and nested logic.
  * **Key Scripts:** **FizzBuzz**, **Code Breaker** (attempt-limited logic), and **Text Adventure** game engines.

## 🛠️ Installation & Usage

To explore the quests within this repository, ensure you have Python 3.x installed.

1.  **Clone the repository:**

    ```bash
    git clone https://github.com/Pirate13Thebes/0xDEAD.git
    cd 0xDEAD
    ```

2.  **Run a specific quest:**

    ```bash
    # Example: Running the FizzBuzz Challenge
    python3 quests/quest_27_fizzbuzz.py
    ```

## 🛡️ Engineering Standards

To maintain the "Exemplary" status of the **0xDEAD** repo, we strictly adhere to:

  * **Reflective Scribing:** Every script includes inline documentation (`#`) explaining the "why" behind the code.
  * **DRY Principle:** Utilizing functions to minimize code repetition.
  * **Input Validation:** Using type casting (e.g., `int(input())`) to handle user data correctly.
  * **Collaborative Control:** All changes are managed via Git to simulate a real-world production environment.
